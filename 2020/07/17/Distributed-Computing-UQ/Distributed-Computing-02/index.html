Distributed Computing

Architectures of Distributed Systems (continued) && Processes

Learning Objects:

  1. Understand how behaviour of middleware can be modified [Architecture]
  2. Understand a basic model for self-managing distributed systems [Architecture]
  3. Understand the role of processes in distributed systems
    – processes, threads in clients or servers
    – processes across multiple systems
  4. Be able to describe virtualisation models for distributed systems

OUTLINE

• Architecture (continued)
– Adaptability and self-management in DS
• Processes
– Definitions(Y)
– The use of threads in DS(Y)
– Virtualisation in DS

– Clients/Servers

  1. Architecture(Continued)

    1.1 Adaptability and self-management in DS 分布式系统的适应能力以及自我管理能力
    • Role of middleware is to provide some degree of distribution
    transparency

    – Hiding distribution of data, processing and control

    • Middleware may have a particular architectural style, e.g.

    – Object-based (e.g. Common Object Request Broker Architecture -
  CORBA)
  – Event-based (most of middleware built for adaptive, context-aware applications)

    • Middleware should be adaptive to meet requirements of
    various application

    1.2 Using interceptors to adapt control
    • Interceptors change flow of control and allow additional code to be executed
    • Many object-based DS use interceptors to change flow of control in the object invocation

    – Requests (object invocations) can be intercepted
– Messages can be intercepted

  2. Processes - Rationale
    • （非）A process is an instance of a computer program that is being executed
    • DS build virtual processors in software, on top of physical processors
    分布式系统会在软件层面上建立虚拟处理器（processor），在物理处理器之上

  3. Multithreaded Servers
    Model Characteristics Description
    Threads Parallelism,blocking system calls Can do tasks concurrently.

    并行性、阻止系统调用                     A blocking call on one thread does not stop other threads executing.

Single-threaded process No parallelism, blocking system calls Cannot do concurrent tasks.
非并行、阻止系统调用

Finitestate machine Parallelism,nonblocking system calls Has one thread. Spends some time on one task. Saves state.
并行、不阻止系统调用 Then spends time on another task. Saves state. Repeat.
Multiple tasks are being completed, but only part of one at any
given time.

  1. Summary
    • Autonomic computing became trendy due to
    complexity of distributed systems
    自动计算流行的原因是分布式系统的复杂性

    • Processes form basis for communication in DS
    Processes 构成了DS中communication的基础

    • Threads are useful in clients and servers
    线程在服务器和客户端很有用

    • Return of resource virtualisation approaches

• Client and servers need to support distribution
transparency
客户端和服务器端需要支持分布透明性
