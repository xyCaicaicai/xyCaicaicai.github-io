  1. Architecture(Continued)

    1.1 Adaptability and self-management in DS 分布式系统的适应能力以及自我管理能力
    • Role of middleware is to provide some degree of distribution
    transparency

    – Hiding distribution of data, processing and control

    • Middleware may have a particular architectural style, e.g.

    – Object-based (e.g. Common Object Request Broker Architecture -
  CORBA)
  – Event-based (most of middleware built for adaptive, context-aware applications)

    • Middleware should be adaptive to meet requirements of
    various application

    1.2 Using interceptors to adapt control
    • Interceptors change flow of control and allow additional code to be executed
    • Many object-based DS use interceptors to change flow of control in the object invocation

    – Requests (object invocations) can be intercepted
– Messages can be intercepted

  2. Processes - Rationale
    • （非）A process is an instance of a computer program that is being executed
    • DS build virtual processors in software, on top of physical processors
    分布式系统会在软件层面上建立虚拟处理器（processor），在物理处理器之上

  3. Multithreaded Servers
    Model Characteristics Description
    Threads Parallelism,blocking system calls Can do tasks concurrently.

    并行性、阻止系统调用                     A blocking call on one thread does not stop other threads executing.

Single-threaded process No parallelism, blocking system calls Cannot do concurrent tasks.
非并行、阻止系统调用

Finitestate machine Parallelism,nonblocking system calls Has one thread. Spends some time on one task. Saves state.
并行、不阻止系统调用 Then spends time on another task. Saves state. Repeat.
Multiple tasks are being completed, but only part of one at any
given time.

  1. Summary
    • Autonomic computing became trendy due to
    complexity of distributed systems
    自动计算流行的原因是分布式系统的复杂性

    • Processes form basis for communication in DS
    Processes 构成了DS中communication的基础

    • Threads are useful in clients and servers
    线程在服务器和客户端很有用

    • Return of resource virtualisation approaches

• Client and servers need to support distribution
transparency
客户端和服务器端需要支持分布透明性

Distributed Computing -01：Introduction and Architectures

Distributed Computing

Introduction to Distributed System

Learning Objects:

  • Understand definitions and goals of distributed systems
  • Be able to describe types of distributed systems
  • Be able to describe various architectures of distributed systems

  1. Definitions of Distributed System
    A distributed system organized as middleware which hides differences between particular nodes.

  2. Goals of DS

    2.1 Transparency(hiding distributed) 透明
    System presents itself as a single computer system.
    1) Access:

    Hide difference in data representation and how a resource is accessed

    2) Location

    Hide where a resource is located

    3) Migration

    Hide that a resource may move to another location

    4) Relocation

    Hide that a resource may be moved to another location while in use

    5) Replication

    Hide that resource is replicated

    6) Concurrency 并发

    Hide that a source may be shared by several competitive users

    7) Failure /ˈfeɪljər/

    Hide the failure and recover of a resource

    2.2 Openness 开放性
    1)Interoperability 互相操作

    the extent by which 2 implementations of systems or components can co-exist and work together.

    2)Portability 接口 

    to what extent an application developed for a distributed system A can be executed without modification, on a different distributed system B that has the same interface as A.

    3)Heterogeneity(异构性/非均匀性)

    components of different characteristics and natures can be integrated together through a unified, standardised interface.

    2.3 Scalability 可拓展性
    2.3.1 Dimensions of Scalability 维度拓展

    1) Size scalability: 尺寸
  we can add more users and/or resources to the system without breaking it up and resemble again with more components.

2)  Geographically /ˌdʒiːəˈɡræfɪkli/  scalable: 地理位置
  locations of different components, resources, and its users doesn’t dictate the performance of the system.

3)  Administratively /ədˈmɪnɪstreɪtɪvli/  scalable:  管理
  the system is still scalable even if it spans many independent administrative organisations.

    2.3.2 Scaling techniques 拓展技术

    1) Hiding communication latencies 隐藏通信延迟
  – Asynchronous communication  异步通信

2) Distribution 分布式
  – Split and spread functionality across the system

  – Decentralise algorithms 去中心化算法

    • No machine has complete information about the system state

    • Machines make decisions based only on local information

    • Failure of one machine does not ruin the algorithm

    • There is no implicit assumption that a global clock exists

3) Replication (including caching) 复制
  1. Summary
    • Distributed systems are built for multicomputers and have to
    deal with
    – Heterogeneous hardware, heterogeneous networks and
    heterogeneous operating systems
    //因为分布式系统是多个电脑组合起来的，所以必须面对、处理 “硬件异构”、“网络异构” 以及 “操作系统异构”

• Distributed systems need to be open, be scalable and provide a
variety of transparencies

• Distributed applications may have different architectural styles
(software architecture)

• Software components, their interactions and placement define
system architecture
